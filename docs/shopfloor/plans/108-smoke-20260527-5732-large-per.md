## Testing strategy

This project explicitly has no test suite (per `CLAUDE.md`: *"No tests: this
is a smoke test target, not a tested app. There is no test suite."*). The
only automated quality gates available are the package scripts:

- **Type check** — `pnpm exec tsc`. Catches the `Task.subtasks` field becoming
  required, mismatches between the migration's row shape and the `Task` type,
  and prop-type errors on the new `SubtaskList`. This layer is *required* to
  pass.
- **Lint** — `pnpm lint`. Catches unused imports / variables introduced by the
  new handlers and component. This layer is *required* to pass.
- **Manual smoke verification via `pnpm dev`** at <http://localhost:3000>:
  1. Fresh browser profile (clean IndexedDB): create a task, add two
     subtasks, toggle one, click the parent's status circle — it should reach
     `in-progress` then loop back to `todo` (rollup blocks `done`).
  2. Complete both subtasks, click the parent circle through `in-progress`
     to `done`.
  3. Re-open one subtask: the parent should demote to `in-progress` and lose
     its `completedAt`.
  4. Delete a subtask; reload the page; verify state persisted.
  5. Migration: in DevTools, manually downgrade by deleting the `TodoApp`
     database, re-create it at v1 by checking out `main` briefly and adding
     a task without subtasks, then check out this branch and reload —
     `Application → IndexedDB → TodoApp → tasks` should show every row with
     `subtasks: []`.

Unit, integration, and end-to-end test layers are **not applicable** to this
project — there is no Jest, Vitest, Playwright, or equivalent harness
installed, and `CLAUDE.md` explicitly designates the project as untested.
Introducing a runner would exceed the scope of this issue.

**Per-task TDD steps (1–4) are skipped under the "no test harness available"
exception.** Each task instead lists the type-check / lint / manual smoke
verification commands that apply to its diff.

## Task 2 — Bump IndexedDB to v2 and backfill `subtasks: []`

**Files**

- Modify: `app/db.ts`

**Steps**

1. Open `app/db.ts`.
2. Change the `DB_VERSION` constant from `1` to `2`:

   ```ts
   const DB_VERSION = 2;
   ```

3. Replace the existing `request.onupgradeneeded = (event) => { ... }`
   handler (lines 25–37) with the version-aware handler below. The v1
   creation branch stays exactly as today; a new `event.oldVersion < 2`
   branch performs the cursor backfill inside the version-change
   transaction:

   ```ts
   request.onupgradeneeded = (event) => {
     const target = event.target as IDBOpenDBRequest;
     const db = target.result;
     const tx = target.transaction!;

     if (!db.objectStoreNames.contains(TASKS_STORE)) {
       const taskStore = db.createObjectStore(TASKS_STORE, { keyPath: "id" });
       taskStore.createIndex("category", "category", { unique: false });
       taskStore.createIndex("status", "status", { unique: false });
       taskStore.createIndex("priority", "priority", { unique: false });
       taskStore.createIndex("dueDate", "dueDate", { unique: false });
     }
     if (!db.objectStoreNames.contains(CATEGORIES_STORE)) {
       db.createObjectStore(CATEGORIES_STORE, { keyPath: "id" });
     }

     if (event.oldVersion < 2) {
       const store = tx.objectStore(TASKS_STORE);
       const cursorReq = store.openCursor();
       cursorReq.onsuccess = () => {
         const cursor = cursorReq.result;
         if (!cursor) return;
         const row = cursor.value as Task;
         if (!Array.isArray(row.subtasks)) {
           cursor.update({ ...row, subtasks: [] });
         }
         cursor.continue();
       };
     }
   };
   ```

4. Do not modify any of the `db.addTask` / `db.updateTask` / `db.deleteTask`
   / `db.getTasks` / `db.getTaskById` / `db.getTasksByCategory` /
   `db.addCategory` / `db.getCategories` method bodies. They already pass
   through whatever `Task` shape the caller hands them, and the new required
   field travels with the row without code changes.

**Notes for the implementer**

- The `tx.objectStore(TASKS_STORE)` call inside the `oldVersion < 2` branch
  is safe even on a brand-new database, because the immediately-preceding
  `db.createObjectStore` (when the store was absent) was executed in the
  same versionchange transaction; `tx.objectStore` returns the just-created
  store. The cursor walk then finds zero rows and exits.
- `Array.isArray(row.subtasks)` guards against the (impossible in normal
  flow but cheap to defend against) case where a stale row already carries
  some other shape; the row is only rewritten when needed, so the migration
  is idempotent if it runs twice.
- Do **not** wrap the cursor walk in a `Promise` or `await`; the
  versionchange transaction is synchronous-eventy and the standard cursor
  callback pattern keeps it open until `cursor` is null. The transaction
  commits when the handler returns and all pending requests resolve.

**Verification**

- `pnpm exec tsc` — must show the same residual errors as after Task 1
  (missing `subtasks` on the construction site in `app/page.tsx`) and no new
  errors inside `app/db.ts`.
- `pnpm lint` — must pass cleanly on `app/db.ts`.

**Commit message**

```
feat(db): bump TodoApp to v2 and backfill empty subtasks on existing rows
```

## Task 3 — Initialize `subtasks` in `handleSave` and gate `cycleStatus` on the rollup

**Files**

- Modify: `app/page.tsx`

**Steps**

1. Open `app/page.tsx`.
2. Add a pure helper above the `Home` component (immediately below the
   existing `statusLabel` function, before `export default function Home()`):

   ```ts
   function allSubtasksDone(task: Task): boolean {
     return task.subtasks.every((s) => s.done);
   }
   ```

   Using `Array.prototype.every` on an empty array returns `true`, so a
   task with zero subtasks is treated as "all done" and the cycle behaves
   exactly as today.

3. Inside `handleSave`, locate the `const task: Task = { ... }` literal (line
   71). Add `subtasks: existing?.subtasks ?? [],` as a new field — place it
   on the line after `completedAt: existing?.completedAt,`. The literal must
   end up as:

   ```ts
   const task: Task = {
     id: editingId || Date.now().toString(),
     title: title.trim(),
     description: description.trim() || undefined,
     category,
     priority,
     status: existing?.status ?? "todo",
     dueDate: dueDate || undefined,
     createdAt: existing?.createdAt ?? now,
     updatedAt: now,
     completedAt: existing?.completedAt,
     subtasks: existing?.subtasks ?? [],
   };
   ```

   The form does not collect subtasks at creation time; they are managed
   per-card after the task exists.

4. Replace the existing `cycleStatus` function body (lines 116–128). The
   replacement keeps the same signature and the same `db.updateTask` +
   `setTasks` write path but gates the `done` transition on
   `allSubtasksDone`:

   ```ts
   async function cycleStatus(task: Task) {
     const order: Task["status"][] = ["todo", "in-progress", "done"];
     const i = order.indexOf(task.status);
     let nextStatus = order[(i + 1) % order.length];
     if (nextStatus === "done" && !allSubtasksDone(task)) {
       nextStatus = "todo";
     }
     const now = new Date().toISOString();
     const updated: Task = {
       ...task,
       status: nextStatus,
       completedAt: nextStatus === "done" ? now : undefined,
       updatedAt: now,
     };
     try {
       await db.updateTask(updated);
       setTasks((prev) => prev.map((t) => (t.id === task.id ? updated : t)));
     } catch (e) {
       console.error(e);
     }
   }
   ```

   This is a behavioural change for tasks with at least one open subtask:
   advancing from `in-progress` now cycles back to `todo` instead of moving
   to `done`. Tasks with zero subtasks or with all subtasks done are
   unaffected.

5. Do not remove the existing `next` object literal style elsewhere — it
   only appears inside `cycleStatus` and the replacement above already
   eliminates it. Do not add any new state, props, or imports for this task.

## Task 4 — Subtask handlers (`addSubtask`, `toggleSubtask`, `deleteSubtask`)

**Files**

- Modify: `app/page.tsx`

**Steps**

1. Open `app/page.tsx`.
2. Update the `Task, Category` import on line 4 to also pull in `Subtask`:

   ```ts
   import { Task, Category, Subtask } from "./types";
   ```

3. Inside the `Home` component, immediately after the `cycleStatus` function
   (which Task 3 placed at lines ~116–135), add the three handlers below.
   All three read the current `Task` from local `tasks` state by id, compute
   the next `Task` value, then persist it via `db.updateTask` and update
   local state in the same shape used by every existing handler.

   ```ts
   async function addSubtask(taskId: string, rawTitle: string): Promise<void> {
     const title = rawTitle.trim();
     if (!title) return;
     const target = tasks.find((t) => t.id === taskId);
     if (!target) return;
     const subtask: Subtask = {
       id: Date.now().toString(),
       title,
       done: false,
     };
     const updated: Task = {
       ...target,
       subtasks: [...target.subtasks, subtask],
       updatedAt: new Date().toISOString(),
     };
     try {
       await db.updateTask(updated);
       setTasks((prev) => prev.map((t) => (t.id === taskId ? updated : t)));
     } catch (e) {
       console.error(e);
     }
   }

   async function toggleSubtask(taskId: string, subtaskId: string): Promise<void> {
     const target = tasks.find((t) => t.id === taskId);
     if (!target) return;
     const nextSubtasks = target.subtasks.map((s) =>
       s.id === subtaskId ? { ...s, done: !s.done } : s,
     );
     const now = new Date().toISOString();
     const allDone = nextSubtasks.every((s) => s.done);
     let nextStatus: Task["status"] = target.status;
     let nextCompletedAt: string | undefined = target.completedAt;
     // Inverse of the cycleStatus rollup: re-opening a subtask demotes a done
     // parent so the invariant "done implies all subtasks done" always holds.
     if (target.status === "done" && !allDone) {
       nextStatus = "in-progress";
       nextCompletedAt = undefined;
     }
     const updated: Task = {
       ...target,
       subtasks: nextSubtasks,
       status: nextStatus,
       completedAt: nextCompletedAt,
       updatedAt: now,
     };
     try {
       await db.updateTask(updated);
       setTasks((prev) => prev.map((t) => (t.id === taskId ? updated : t)));
     } catch (e) {
       console.error(e);
     }
   }

   async function deleteSubtask(taskId: string, subtaskId: string): Promise<void> {
     const target = tasks.find((t) => t.id === taskId);
     if (!target) return;
     const updated: Task = {
       ...target,
       subtasks: target.subtasks.filter((s) => s.id !== subtaskId),
       updatedAt: new Date().toISOString(),
     };
     // Deletion intentionally does not auto-promote a parent whose remaining
     // subtasks all happen to be done — consistent with cycleStatus, which
     // also never auto-promotes.
     try {
       await db.updateTask(updated);
       setTasks((prev) => prev.map((t) => (t.id === taskId ? updated : t)));
     } catch (e) {
       console.error(e);
     }
   }
   ```

4. Do not change `addTask` / `updateTask` in `app/db.ts`. Subtask mutations
   ride on the existing whole-task `db.updateTask` round-trip.
5. Do not auto-advance the parent to `done` when the last open subtask is
   toggled. The spec is explicit: the rollup is a one-way constraint, not
   an auto-promoter. The only automatic status change is the
   `done → in-progress` demotion in `toggleSubtask`.

**Verification**

- `pnpm exec tsc` — must pass cleanly. The new handlers are unreferenced for
  now (the UI wires them up in Task 5); TypeScript does not error on unused
  local functions inside a component.
- `pnpm lint` — must pass. If the project's ESLint config flags unused
  symbols (the default Next.js config does not, but a stricter override
  might), confirm before making any change; if it does fire, leave the
  handlers in place — Task 5 will reference them within the same PR.

**Commit message**


   ```tsx
   function SubtaskList(props: {
     task: Task;
     onAdd: (title: string) => void;
     onToggle: (subtaskId: string) => void;
     onDelete: (subtaskId: string) => void;
   }) {
     const { task, onAdd, onToggle, onDelete } = props;
     const [draft, setDraft] = useState("");

     function submit() {
       const value = draft.trim();
       if (!value) return;
       onAdd(value);
       setDraft("");
     }

     return (
       <div
         style={{
           marginTop: "0.75rem",
           marginLeft: "2.25rem",
           display: "flex",
           flexDirection: "column",
           gap: "0.375rem",
         }}
       >
         {task.subtasks.map((s) => (
           <div
             key={s.id}
             style={{
               display: "flex",
               alignItems: "center",
               gap: "0.5rem",
             }}
           >
             <button
               type="button"
               onClick={() => onToggle(s.id)}
               title={s.done ? "Mark subtask incomplete" : "Mark subtask done"}
               style={{
                 flexShrink: 0,
                 width: 16,
                 height: 16,
                 borderRadius: "50%",
                 border: `2px solid ${s.done ? "var(--accent-gold)" : "var(--border-hover)"}`,
                 background: s.done ? "var(--accent-gold)" : "transparent",
                 color: s.done ? "#0c1524" : "transparent",
                 cursor: "pointer",
                 display: "flex",
                 alignItems: "center",
                 justifyContent: "center",
                 fontSize: "0.6rem",
                 fontWeight: 700,
                 lineHeight: 1,
               }}
             >
               {s.done ? "✓" : ""}
             </button>
             <span
               style={{
                 flex: 1,
                 fontSize: "0.8125rem",
                 color: s.done ? "var(--text-muted)" : "var(--text-primary)",
                 textDecoration: s.done ? "line-through" : "none",
                 overflow: "hidden",
                 textOverflow: "ellipsis",
                 whiteSpace: "nowrap",
               }}
             >
               {s.title}
             </span>
             <button
               type="button"
               className="btn btn-ghost btn-sm"
               onClick={() => onDelete(s.id)}
               title="Delete subtask"
               style={{ padding: "0 0.5rem" }}
             >
               ×
             </button>
           </div>
         ))}
         <input
           className="input"
           type="text"
           placeholder="Add subtask…"
           value={draft}
           onChange={(e) => setDraft(e.target.value)}
           onKeyDown={(e) => {
             if (e.key === "Enter") {
               e.preventDefault();
               submit();
             }
           }}
           style={{ fontSize: "0.8125rem" }}
         />
       </div>
     );
   }
   ```
