# Plan: Bulk-select and delete tasks

Issue: niranjan94/shopfloor-smoke#105 — `smoke-20260527-5732/medium: bulk-select and delete tasks`

No design spec exists for this issue (medium-complexity flow). The design below is derived from the issue body and triage comment.

## Goal

Add a bulk-select mode to the tasks list on the home page. When enabled, every visible task card shows a checkbox; a "Delete selected" button at the top of the task list removes all checked tasks via `db.deleteTask` and updates the React `tasks` state. Selection state is local React state and resets when bulk-select is toggled off.

## Scope and constraints

In scope:

- `app/page.tsx` — wire bulk-select mode, selection set, and delete handler.
- One new file under `app/components/` — a thin task-card component that knows how to render the optional checkbox. The exact name is fixed in Task 2.

Out of scope (do NOT touch in this plan):

- `app/db.ts`, `app/types.ts`, `app/layout.tsx`, sidebar, or stub pages.
- Any IndexedDB schema change. The plan uses the existing `db.deleteTask(id)` function at `app/db.ts:64`.
- Persisting selection or bulk-select mode across reloads — state is local React state only.
- Bulk operations other than delete (status change, category move, etc.).

Constraints from the issue:

- Touch `app/page.tsx` and at most one new component file under `app/components/`.
- No schema changes, no new persistence fields.
- Selection state resets when bulk-select is toggled off.

## Task 1 — Add bulk-select state to `app/page.tsx`

**Affected files**

- Modify: `app/page.tsx`

**Behavior introduced**

The home component declares two new React state variables that are not yet referenced by JSX:

- `bulkSelectMode: boolean`, initial value `false`.
- `selectedIds: Set<string>`, initial value `new Set<string>()`.

Toggling `bulkSelectMode` off must clear `selectedIds`. Provide a single helper `toggleBulkSelectMode` that flips the boolean and, when turning off, resets the set to a new empty `Set<string>()`.

Also add two helpers used in later tasks:

- `toggleSelected(id: string)` — returns a new `Set<string>` derived from the current `selectedIds` with `id` added if absent or removed if present, then calls `setSelectedIds` with that new set. Always allocate a new `Set` (do not mutate the existing one) so React detects the change.
- `clearSelection()` — calls `setSelectedIds(new Set<string>())`.

**Implementation detail**

Add the state and helpers inside `Home()` in `app/page.tsx`, immediately after the existing `loading` state declaration on line 47 and before `useEffect(() => { loadData(); }, []);` on line 49. Use this exact code:

```tsx
const [bulkSelectMode, setBulkSelectMode] = useState(false);
const [selectedIds, setSelectedIds] = useState<Set<string>>(new Set());

function toggleBulkSelectMode() {
  setBulkSelectMode((prev) => {
    const next = !prev;
    if (!next) setSelectedIds(new Set());
    return next;
  });
}

function toggleSelected(id: string) {
  setSelectedIds((prev) => {
    const next = new Set(prev);
    if (next.has(id)) next.delete(id);
    else next.add(id);
    return next;
  });
}

function clearSelection() {
  setSelectedIds(new Set());
}
```

No JSX changes in this task; the helpers are wired up in Tasks 3 and 4.

**Verification**

- Run `pnpm exec tsc --noEmit` — must pass.
- Run `pnpm lint` — must pass. If `lint` complains about `clearSelection` being unused, leave the helper in place; it is consumed in Task 4. If lint is configured to error on unused vars, prefix-suppress with an `eslint-disable-next-line` comment on the `function clearSelection` line citing the rule it triggers; do not delete the helper.

```tsx
"use client";

import { Task } from "../types";

export interface TaskCardProps {
  task: Task;
  index: number;
  categoryName: string;
  bulkSelectMode: boolean;
  selected: boolean;
  onToggleSelected: (id: string) => void;
  onCycleStatus: (task: Task) => void;
  onEdit: (task: Task) => void;
  onDelete: (id: string) => void;
}

function priorityClass(p: string) {
  if (p === "high") return "task-card task-card-high";
  if (p === "medium") return "task-card task-card-medium";
  return "task-card task-card-low";
}

function statusBadgeClass(s: string) {
  if (s === "todo") return "badge badge-todo";
  if (s === "in-progress") return "badge badge-doing";
  return "badge badge-done";
}

function statusLabel(s: string) {
  if (s === "todo") return "To Do";
  if (s === "in-progress") return "In Progress";
  return "Done";
}

export function TaskCard({
  task,
  index,
  categoryName,
  bulkSelectMode,
  selected,
  onToggleSelected,
  onCycleStatus,
  onEdit,
  onDelete,
}: TaskCardProps) {
  return (
    <div
      className={priorityClass(task.priority)}
      style={{ animation: `slideInUp 350ms ease-out ${index * 30}ms both` }}
    >
      <div style={{ display: "flex", alignItems: "flex-start", gap: "0.875rem" }}>
        {bulkSelectMode && (
          <input
            type="checkbox"
            checked={selected}
            onChange={() => onToggleSelected(task.id)}
            aria-label={`Select task ${task.title}`}
            style={{ flexShrink: 0, marginTop: "4px", width: 18, height: 18, cursor: "pointer" }}
          />
        )}
        <button
          onClick={() => onCycleStatus(task)}
          title={`Status: ${statusLabel(task.status)} — click to advance`}
          style={{
            flexShrink: 0,
            marginTop: "2px",
            width: 22,
            height: 22,
            borderRadius: "50%",
            border: `2px solid ${task.status === "done" ? "var(--accent-gold)" : task.status === "in-progress" ? "#fcd34d" : "var(--border-hover)"}`,
            background: task.status === "done" ? "var(--accent-gold)" : task.status === "in-progress" ? "rgba(252, 211, 77, 0.15)" : "transparent",
            color: task.status === "done" ? "#0c1524" : "transparent",
            cursor: "pointer",
            display: "flex",
            alignItems: "center",
            justifyContent: "center",
            fontSize: "0.7rem",
            fontWeight: 700,
            transition: "all 150ms ease",
          }}
        >
          {task.status === "done" ? "✓" : task.status === "in-progress" ? "●" : ""}
        </button>

        <div style={{ flex: 1, minWidth: 0 }}>
          <div style={{
            fontWeight: 600,
            color: task.status === "done" ? "var(--text-muted)" : "var(--text-primary)",
            textDecoration: task.status === "done" ? "line-through" : "none",
            marginBottom: task.description ? "0.25rem" : "0.5rem",
          }}>
            {task.title}
          </div>
          {task.description && (
            <div style={{
              fontSize: "0.8125rem",
              color: "var(--text-muted)",
              marginBottom: "0.5rem",
              overflow: "hidden",
              textOverflow: "ellipsis",
              whiteSpace: "nowrap",
            }}>
              {task.description}
            </div>
          )}
          <div style={{ display: "flex", gap: "0.375rem", flexWrap: "wrap" }}>
            <span className="badge badge-category">{categoryName}</span>
            <span className={statusBadgeClass(task.status)}>{statusLabel(task.status)}</span>
            <span className={`badge badge-${task.priority}`}>
              {task.priority.charAt(0).toUpperCase() + task.priority.slice(1)}
            </span>
            {task.dueDate && (
              <span className="badge" style={{ background: "rgba(77, 184, 168, 0.12)", color: "var(--accent-teal)" }}>
                {new Date(task.dueDate + "T00:00:00").toLocaleDateString(undefined, { month: "short", day: "numeric" })}

            )}

        </div>

        <div style={{ display: "flex", gap: "0.375rem", flexShrink: 0 }}>
          <button className="btn btn-ghost btn-sm" onClick={() => onEdit(task)}>
            Edit
          </button>
          <button className="btn btn-danger btn-sm" onClick={() => onDelete(task.id)}>
            Delete
          </button>

    </div>
```

## Task 4 — Add bulk-select toolbar and bulk delete handler

**Affected files**

- Modify: `app/page.tsx`

**Behavior introduced**

A toolbar in the task list header that:

- Always shows a "Bulk select" toggle button. Clicking it flips `bulkSelectMode` via the `toggleBulkSelectMode` helper from Task 1.
- When `bulkSelectMode` is `true`, additionally shows:
  - The current selection count, e.g. `3 selected`.
  - A "Delete selected" button. Disabled when `selectedIds.size === 0`. When clicked, it deletes every task whose id is in `selectedIds` via `db.deleteTask`, removes those tasks from React `tasks` state, clears the selection, and leaves `bulkSelectMode` enabled (so the user can continue selecting). Errors from `db.deleteTask` are caught and logged via `console.error`, matching the existing single-delete handler at `app/page.tsx:95-100`.
  - A "Cancel" button that turns off bulk-select mode (which, per Task 1, also clears selection).

When `bulkSelectMode` is `false`, the toolbar shows only the "Bulk select" button and the existing task-count header on the left.

**Implementation**

1. Add a `handleBulkDelete` async function inside `Home()`, placed immediately after `handleDelete` (currently `app/page.tsx:95-100`):

   ```tsx
   async function handleBulkDelete() {
     const ids = Array.from(selectedIds);
     if (ids.length === 0) return;
     try {
       await Promise.all(ids.map((id) => db.deleteTask(id)));
       setTasks((prev) => prev.filter((t) => !selectedIds.has(t.id)));
       clearSelection();
     } catch (e) { console.error(e); }
   }
   ```

   Use `Promise.all` (not a sequential `for` loop) — each `db.deleteTask` opens its own IndexedDB transaction in `app/db.ts:64-73`, so they are safe to run concurrently.

2. Replace the task-list header block at `app/page.tsx` lines 279–281:

   ```tsx
   <div style={{ display: "flex", alignItems: "center", justifyContent: "space-between", marginBottom: "0.875rem" }}>
     <h2>{filtered.length} {filtered.length === 1 ? "Task" : "Tasks"}</h2>
   </div>
   ```

   with:

   ```tsx
   <div style={{ display: "flex", alignItems: "center", justifyContent: "space-between", marginBottom: "0.875rem", gap: "0.75rem", flexWrap: "wrap" }}>
     <h2>{filtered.length} {filtered.length === 1 ? "Task" : "Tasks"}</h2>
     <div style={{ display: "flex", alignItems: "center", gap: "0.5rem" }}>
       {bulkSelectMode && (
         <>
           <span style={{ color: "var(--text-muted)", fontSize: "0.875rem" }}>
             {selectedIds.size} selected
           </span>
           <button
             className="btn btn-danger btn-sm"
             onClick={handleBulkDelete}
             disabled={selectedIds.size === 0}
           >
             Delete selected
           </button>
           <button className="btn btn-muted btn-sm" onClick={toggleBulkSelectMode}>
             Cancel
           </button>
         </>
       )}
       {!bulkSelectMode && (
         <button className="btn btn-ghost btn-sm" onClick={toggleBulkSelectMode}>
           Bulk select
         </button>
       )}
     </div>
   </div>
   ```

   The `btn btn-muted` and `btn btn-ghost` class combinations already exist elsewhere in `page.tsx` (e.g. lines 234, 359) — no CSS change is needed.

**Verification**

- Run `pnpm exec tsc --noEmit` — must pass.
- Run `pnpm lint` — must pass.
- Run `pnpm build` — must succeed.
- Run `pnpm dev` and manually walk through the following in the browser at `http://localhost:3000`:
  1. With at least 3 tasks present, the task list header shows `N Tasks` on the left and a `Bulk select` button on the right. No checkboxes appear on task cards.
  2. Click `Bulk select`. The button is replaced by `0 selected`, a disabled `Delete selected` button, and a `Cancel` button. Each task card now shows a checkbox on its left.
  3. Tick two checkboxes. The count updates to `2 selected`. `Delete selected` becomes enabled.
  4. Click `Delete selected`. Both selected tasks disappear from the list. The count returns to `0 selected`. `Delete selected` becomes disabled again. `bulkSelectMode` is still on (checkboxes remain visible).
  5. Reload the page (Cmd+R / Ctrl+R). The deleted tasks remain deleted (IndexedDB persisted). Bulk-select mode resets to off (state is local).
  6. Click `Bulk select`, tick one task, click `Cancel`. Bulk-select mode is off, checkboxes disappear, the unchecked tasks remain.
  7. Single-task `Edit` and `Delete` buttons on each card still work in non-bulk mode. Status toggle still cycles.
